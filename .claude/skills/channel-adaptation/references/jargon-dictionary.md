# Jargon Dictionary - Technical Term Replacements

Quick reference for replacing technical jargon with user-friendly language.

## Core Replacements

| ❌ Technical Term | ✅ User-Friendly Alternative |
|------------------|----------------------------|
| backend | system |
| endpoint | connection point / portal |
| database | storage / records |
| API | integration / connection |
| webhook | notification system |
| authentication | login / access |
| authorization | permission |
| cache | temporary storage |
| deploy | publish / launch |
| repository | project folder |
| server | system |
| client | your device / your app |
| query | search / request |
| schema | structure / layout |
| migration | update / transfer |
| rollback | undo / revert |
| latency | delay / response time |
| throughput | speed / capacity |
| payload | data / information |
| token | access code / key |

## Context-Aware Usage

**Exception**: Keep technical terms if customer used them first (mirror their language).

**Example**:
- Customer says: "My API endpoint is failing"
- Response: "I see your API endpoint is returning errors..." ✅
- Customer says: "My connection isn't working"
- Response: "I see your connection is having issues..." ✅ (don't say "endpoint")

## Creative Industry Specific

For CloudStream CRM's creative agency audience:

| ❌ Avoid | ✅ Use Instead |
|---------|---------------|
| "Execute the script" | "Run the automation" |
| "Compile the assets" | "Prepare the files" |
| "Render the output" | "Generate the preview" |
| "Parse the data" | "Process the information" |
| "Sync the repository" | "Update the project" |
